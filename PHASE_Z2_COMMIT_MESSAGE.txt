Phase Z2: Close alignment misses via verified CSV mappings (Part 1/2)

Implements CSV ingestion and validation tools for Phase Z2 miss reduction initiative.
Target: Reduce 54 unique misses to ≤10 through verified FDC mappings, normalization
fixes, and systematic ignore rules.

## This Commit (Part 1/2 - Tools & Documentation)

### New Tools
- tools/merge_verified_fallbacks.py (636 lines)
  * Parses missed_food_names.csv (104 rows → 54 unique foods)
  * DB validation with precedence rules (protects verified entries)
  * Kcal inference for produce, proteins, grains, oils
  * Special case handling: cherry tomato, chicken, chilaquiles, orange w/ peel
  * Outputs: verified YAML + merged config + JSON report

- tools/validate_stageZ_config.py (304 lines)
  * Validates duplicate keys, kcal ranges, FDC IDs, synonym conflicts
  * Summary table output with DB verification status
  * CI/CD compatible (exit code 1 on critical errors)

### Documentation
- PHASE_Z2_README.md - User-facing implementation guide
- docs/phase_z2_implementation_status.md - Technical deep-dive
- PHASE_Z2_SUMMARY.md - Executive summary

### Key Features
- **Precedence Enforcement**: DB-verified entries protected from CSV overwrites
- **Robust Error Handling**: Malformed CSV rows logged & skipped, don't fail merge
- **Kcal Inference**: Auto-calculates ranges when CSV lacks values
- **Validation Suite**: Comprehensive checks for config integrity
